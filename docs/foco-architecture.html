技術棧選型

根據 FOCO 的 3D 寵物、計時器、動畫、跨平台需求選定

📱
Expo SDK 52
托管工作流（Managed Workflow），支援 EAS Build / OTA 更新。無需觸碰原生代碼即可使用相機、通知、GL 等原生模組。
🗂
Expo Router 4.x
File-based routing，對應 app/ 目錄自動生成路由。支援 (groups)、動態路由 [id]、共享 _layout。
🐻
Zustand 4.x
輕量全域狀態管理。比 Redux 簡潔，比 Context 效能佳。用於 authStore / profileStore / sessionStore / statsStore。
💾
MMKV react-native-mmkv
高效能本地持久化（比 AsyncStorage 快 10x）。搭配 Zustand persist middleware 儲存 user/pet 狀態。
🎭
Reanimated 3.x
UI Thread 原生動畫，60fps。用於寵物浮動動畫、頁面過渡、XP 條填充、成就爆炸粒子效果。
🌐
expo-three + three.js
取代 Web 版的 Three.js Canvas。搭配 expo-gl 渲染 XingWang3D 3D 寵物角色，支援手勢旋轉 / 縮放。
🎨
NativeWind 4.x
Tailwind CSS 語法用於 React Native。原型設計快速，減少 StyleSheet boilerplate，對應現有 className 體系。
🔥
Supabase 後端即服務
Auth（Email / Apple / Google SSO）、PostgreSQL 資料庫、Edge Functions。對應 ScreenSignup 的三種登入方式。
🔔
expo-notifications Expo
對應 Consent 畫面的「Allow gentle focus reminders」。排程推播提醒用戶開始專注 session。
📊
Victory Native XL 圖表
用 Skia 渲染高效能圖表。對應 Stats 頁的甜甜圈（時間分類）、長條圖（每日專注）、進度條（干擾源）。
🤝
react-native-gesture-handler Expo
原生手勢識別，搭配 Reanimated。用於寵物 3D 旋轉（Pan）、Backpack 橫向滑動（Swipe）。
🛡
TypeScript 5.x strict
strict mode 開啟，確保 store / API 型別安全。搭配 Zod 做執行期 API response 驗證。

狀態管理設計

Zustand + MMKV persist — 4 個獨立 store，對應畫面原始 state 物件

authStore auth.ts · persist
email string ●
password string ●
isLoggedIn boolean ●
sessionToken string | null ○
onboardingDone boolean ●
profileStore profile.ts · persist
nickname string ●
role 'Student' | 'Working pro' | 'Creator' | 'Other' ●
pains string[] ○
pet 'sprout' | 'ember' | 'mist' | 'pebble' ●
tos boolean ●
privacy boolean ●
marketing boolean ○
notifs boolean ●
sessionStore session.ts · 不持久化
activeMissionId string | null ○
timerSeconds number ●
isRunning boolean ●
distractionCount number ●
earnedXP number ○
rewards RewardItem[] ○
statsStore stats.ts · persist
totalXP number ●
level number ●
streak number ●
totalFocusMinutes number ●
dailyFocusLog DayLog[] ●
categoryBreakdown Record<string, number> ●
backpackItems BackpackItem[] ●
