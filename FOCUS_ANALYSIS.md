# FOCO 專注分析邏輯 — 現況 × 優化思考

> 目標：從「記錄工具」進化成「真正理解使用者專注習慣的夥伴」

---

## 一、現有系統架構

### 目前收集的原始資料

每次專注結束，系統記錄以下數據：

| 欄位 | 說明 |
|---|---|
| `planned_duration` | 使用者設定的目標時長 |
| `actual_duration` | 實際專注秒數 |
| `pause_count` | 暫停次數 |
| `pause_total_sec` | 總暫停秒數 |
| `left_app_count` | 切出 App 次數 |
| `left_app_total_sec` | 切出 App 總秒數 |
| `completed` | 是否完成目標時長 |
| `early_stop` | 是否提前結束 |
| `started_at` | 開始時間（含時區） |

### 目前的分析輸出

1. **DISC 專注類型**（每次 session → Edge Function 回傳）
   - Dominance 主導型 / Influence 影響型 / Steadiness 穩健型 / Conscientiousness 謹慎型
2. **週統計**：總時數、session 數、連續天數（streak）
3. **每日折線圖**：一週專注時長視覺化
4. **DISC 雷達圖**：4 個向度的長期分佈
5. **結束後報告**：時長 + 暫停 + 切出 + DISC 類型 + XP

## 二、現有系統的問題

### 2.1 DISC 分類邏輯不透明

使用者看到「你是 Conscientiousness 謹慎型」，但完全不知道為什麼。分類依據沒有解釋，導致：
- 無法從報告中學到任何行為調整方向
- DISC 標籤感覺像隨機貼貼紙，而非有意義的回饋

### 2.2 XP 只反映時間，不反映品質

`xp_gained` 目前幾乎等於「你撐了多久」，沒有考量：
- 是否完成任務
- 中途有沒有被打擾
- 相同時長但不同行為品質的差異

這讓 XP 失去激勵效果——使用者不會覺得「我今天表現特別好」。

### 2.3 缺乏時間模式分析

`started_at` 有紀錄，但從未被分析。使用者完全不知道：
- 自己在哪個時段最能進入深度專注
- 是否有週期性的高/低效率日

### 2.4 任務類型與專注品質沒有關聯

任務只有標題和時長，沒有分類標籤。無法回答：
- 我做哪類任務比較容易分心？
- 哪類任務完成率最高？

### 2.5 寵物成長與使用者行為脫節

寵物升級靠 XP 累積，但：
- 寵物反應的是「你用了多久 app」，不是「你的專注有多深」
- 使用者沒有辦法覺得「我今天的表現讓寵物變得更好」

---

## 三、使用者真正需要什麼

> 從使用者角度出發，好的專注分析應該回答三個問題：

**① 我今天做得怎麼樣？**
→ 不只是時間，要有品質感

**② 我最適合什麼時候、用什麼方式專注？**
→ 個人化模式，越用越準

**③ 我有在進步嗎？**
→ 有意義的成長感，不只是數字增加

---

## 四、優化方向

### 4.1 Session 品質分數（Quality Score）

**現況**：只有 completed / not completed  
**改進**：計算 0–100 的品質分，作為 XP 係數

```
基礎分  = min(actual / planned, 1.0) × 60     // 完成度
暫停扣分 = pause_count × 5 (最多扣 20)
切出扣分 = left_app_count × 8 (最多扣 30)
完成獎勵 = completed ? +20 : 0

Quality Score = max(0, 基礎分 - 暫停扣分 - 切出扣分 + 完成獎勵)
XP = base_xp × (0.5 + Quality × 0.5 / 100)   // 最低 50%，最高 150%
```

使用者可以感受到「這次我很專注，XP 特別多」。

---

### 4.2 讓 DISC 類型有解釋

每次 session 的 DISC 分類加上一句「為什麼」：

| 情境 | DISC 類型 | 說明文字 |
|---|---|---|
| 完成目標、幾乎沒打擾 | Conscientiousness | 你今天零干擾完成目標，完美執行力 |
| 完成目標、有暫停但堅持 | Dominance | 雖然中途暫停，你依然完成了。目標感強 |
| 沒完成、但持續了很久 | Steadiness | 沒跑完全程，但你很有耐心地堅持 |
| 多次切出、短暫完成 | Influence | 今天分心比較多，嘗試縮短下次時段 |

---

### 4.3 最佳專注時段分析

利用已有的 `started_at` 欄位，累積足夠資料後（≥10 次）顯示：

```
你的黃金時段

早上 9–11 點
平均 Quality Score：82 / 100
完成率：90%

vs 下午 3–5 點
平均 Quality Score：61 / 100
完成率：65%
```

**建議**：超過 10 次 session 後，在設定時間那一步提示「你通常在這個時段更專注」。

---

### 4.4 任務分類標籤

在 FocusSetupModal 中讓使用者選擇任務類型（選填）：

- 📚 讀書複習
- ✍️ 寫作 / 作業
- 💻 寫程式
- 🎨 創作
- 📋 行政雜事

長期後可以分析：「你在做創作類任務時完成率最高」。

---

### 4.5 寵物情緒與專注品質連動

目前寵物只會長大（level up），不會有即時反應。  
改為讓寵物狀態反映最近表現：

| 最近 3 次平均 Quality | 寵物狀態 | 顯示 |
|---|---|---|
| ≥ 80 | 精神飽滿 | 動作更活潑、idle 動畫加速 |
| 50–79 | 一般 | 正常動畫 |
| < 50 | 有點累 | 動作放慢、偶爾打哈欠動畫 |

讓使用者感受到「我的專注行為直接影響我的寵物」。

---

### 4.6 週目標與進度感

在首頁或 Stats 加入週目標設定：

```
本週目標：10 小時
目前進度：6.5 / 10 小時 ████████░░ 65%
還差 3.5 小時，剩 3 天，每天約需 1.2 小時
```

讓使用者有明確的前進方向，而不是被動看數字。

---

### 4.7 洞察敘事（Insight Narrative）

Stats 頁底部加一段每週自動生成的文字描述：

> 「這週你完成了 8 次專注，比上週多 3 次。  
> 平均品質分是 74 分，最好的一次是週三早上的 50 分鐘讀書。  
> 你在早上的表現持續比下午好——考慮把重要任務排在上午！」

不需要 AI，用 rule-based 模板就可以產生有感的文字。

## 五、資料擴充建議

目前 `sessions` 表建議新增：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `quality_score` | integer 0–100 | session 品質分數 |
| `task_category` | text nullable | 任務類型標籤 |
| `mood_before` | integer 1–5 nullable | 開始前主觀狀態 |
| `mood_after` | integer 1–5 nullable | 結束後主觀狀態 |
| `notes` | text nullable | 使用者自由備註 |

`mood` 欄位是選填的，讓願意的使用者建立「情緒 × 專注」的個人資料庫。

## 六、優化優先順序

| 優先級 | 功能 | 難度 | 衝擊 |
|---|---|---|---|
| 🔴 高 | Quality Score + XP 品質係數 | 低（Edge Function 改動）| 高 |
| 🔴 高 | DISC 分類加解釋文字 | 低（前端 copy）| 高 |
| 🟡 中 | 週目標設定 | 低（前端）| 中 |
| 🟡 中 | 洞察敘事（rule-based）| 中（前端邏輯）| 高 |
| 🟡 中 | 寵物情緒連動 | 中（動畫 + 狀態）| 高 |
| 🟢 低 | 最佳時段分析 | 中（需累積資料）| 高（長期）|
| 🟢 低 | 任務分類標籤 | 中（UI + DB）| 中 |
| 🟢 低 | mood 追蹤 | 高（需要習慣養成）| 不確定 |

## 七、核心設計原則

1. **不要只給數字，要給意義** — 「你專注了 47 分鐘」不如「你今天完成率 94%，比你的平均高 12%」
2. **讓使用者感受到進步** — 每次結束要有一個「今天做得好的地方」
3. **個人化越早越好** — 第一週就要開始學習使用者的模式，不要等資料夠多才顯示
4. **寵物是情感錨點** — 所有分析回饋都要與寵物連結，讓數字有情感溫度
5. **不要讓分析變成壓力** — 建議語氣要鼓勵，不要評判

---

*最終目標：使用者打開 Stats 不是在看自己的數據，而是在和 FOCO 一起回顧今天。*
